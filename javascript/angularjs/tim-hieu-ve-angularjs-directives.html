Tìm hiểu về AngularJS Directives

Oct 24, 2014
8 minute read

Directives là gì?

Các thuộc tính, id, class, name, onClick, … của một DOM element được gọi chung là directives. AngularJS sẽ dựa vào những directive đó để đính kèm các chỉ thị hoặc các sự kiện tới DOM element cần xử lý, thậm chí là thay đổi cấu trúc của DOM.

AngularJS có sẵn một bộ directives: ng-app, ng-module, ng-controller, … Để phát triển ứng dụng với AngularJS thì cần phải nắm rõ cách hoạt động và mục đích sử dụng của những directive đó. Trong phạm vi bài viết này sẽ không giải thích và hướng dẫn cách sử dụng những directive sẵn có của AngularJS.

Cách directives được biên dịch

Khi một trang được load thì web brower sẽ tự động parses HTML vào DOM. Việc biên dịch HTML được thực thi trong 3 bước:

  1. $compile sẽ quét qua các DOM elements, nếu compiler tìm thấy element nào có khai báo directive, thì directive tương ứng sẽ được thêm vào danh sách các directives phù hợp với DOM element. Một element có thể khai báo nhiều directives.
  2. Khi một DOM element khai báo nhiều directives, compiler sẽ sắp xếp thứ tự biên dịch các directives theo thuộc tính priority của mỗi directive. Compiler functions của mỗi directive sẽ được thực thi. Mỗi compiler function có thể làm thay đổi DOM. Mỗi compiler function phải trả về một link function, xử lý liên kết giữa DOM và scope.
  3. $compile liên kết template với scope bằng cách gọi linking function đã liên kết từ bước 2. Đăng ký theo dõi các elements và thiết lập cơ chế $watchs với scope của directive được cấu hình.

Sau khi thực thi biên dịch HTML thì giữa scope và DOM sẽ có sự ràng buộc (a live binding), một thay đổi ở model trên compiled scope sẽ được ánh xạ vào DOM.

Directive’s scope

Mặc định thì dỉective sẽ lấy scope cha bên ngoài, nhưng trong một số truờng hợp chúng ta muốn directive khai báo thêm các thuộc tính và function khác chỉ sử dụng bên trong nội bộ directive, mà lại không làm ảnh huởng đến scope cha bên ngoài thì có 2 lựa chọn:

  • A child scope: scope này thừa kế từ scope cha bên ngoài, và có thể được khai báo thêm các property, function khác.
  • An isolated scope: là một scope riêng mới, không thừa kế gì từ scope bên ngoài, chỉ tồn tại mỗi mình nó.

Sử dụng isolate scope cho truờng hợp khi chúng ta tạo một custom directive, và muốn tái sử dụng directive đó.

Sử dụng isolate scope không có nghĩa là không truy cập đuợc vào scope cha bên ngoài.

Tạo custom directive

Tên của Angular Directives được khai báo theo kiểu camelCase.

angular.module('myDemoModule', [])
  .directive('myDemoDirective', function(){
	   return {
	// TODO
   };
});

Một mẫu định nghĩa directive

var myDemoModule = angular.module('myDemoModule', []);
myDemoModule.directive(‘myDemoDirective’, function factory(injectables) {
  return {
    restrict: "AECM”, // kiểu string
    priority: 3, // kiểu number,
    template: "<p></p>", // kiểu string
    templateUrl: "myDirectivesFolder / demoTempDirective.html”, //kiểu string
    replace: true, // kiểu boolean
    transclude: false, // kiểu boolean
    scope: {
      title: '@'
    }, // kiểu bool hoặc object
    controller: function controllerConstructor($scope, $element, $attrs, $transclude) {…},
    require: ['^myTabs', '^myModel'], // kiểu string
    link: function postLink(scope, iElement, iAttrs) {…},
    compile: function compile(tElement, tAttrs, transclude) {
      return {
        pre: function preLink(scope, iElement, iAttrs, controller) {…},
        post: function postLink(scope, iElement, iAttrs, controller) {…}
      }
    }
  };
});

Các thuộc tính khai báo

restrict

khai báo kiểu directive. Có các giá trị (A, E, C, M) tương ứng 4 kiểu directive.

  1. Kiểu attribute (A)
    <div my-directive></div>
  2. Kiểu element (E)
    <my-directive></my-directive>
  3. Kiểu class (C)
    <div class=”my-directive”></div>
  4. Kiểu comment (M)
    <!-- directive: my-directive -->

VD:
index.html

<div ng-app="myDemoModule">
	<div ng-controller="demoController">  
	    <ANY my-demo-directive><ANY>
  	    <my-demo-directive></my-demo-directive>
	    <ANY class="my-demo-directive"></ANY>
	    <!-- directive: my-demo-directive -->
	</div>
</div>

script.js

angular.module('myDemoModule', [])
      .directive('myDemoDirective', function(){
    	   return {
		restrict: 'A',
		template: '<p>This is my Angular directive demo</p>'
	   };
});
priority

Thứ tự thực thi directive.

Khi một DOM element khai báo nhiều directive thì cần phân định rõ thứ tự thực thi directive, vì vậy priority được dùng để sắp xếp các directive trước khi compile function được gọi. Priority được khai báo kiểu number, theo thứ tự thực thi 0 → 1 → 2 → … → n (Giá trị mặc định của priority là “0”).

Trường hợp các directive có giá trị priority bị trùng nhau thì trình biên dịch sẽ thực thi directive được khai báo trước.

Các “pre-link” function cũng được thực thi theo thứ tự của priority, còn “post-link” thì được thực thi theo thứ tự ngược lại.

VD:
Các directive02, directive01, directive03 đều khai báo “priority: 1”

  • <p directive02 directive01 directive03 ></p>
    Thứ tự thực thi sẽ là: directive02 → directive01 → directive03.
  • <p directive03 directive01 directive02 ></p>
    Thứ tự thực thi sẽ là: directive03 → directive01 → directive02.

Nếu các directive đó đều có tác động đến việc hiển thị của DOM element, thì trình biên dịch sẽ ưu tiên xử lý của directive có giá trị priority nhỏ nhất.

template

Định nghĩa template cho directive của bạn (không sử dụng nếu dùng templateURL).

VD: template: “<p>This is my demoTemplate</p>”

templateUrl

Khai báo đường dẫn (Url) của file định nghĩa template được dùng cho directive. Được sử dụng khi template của directive phức tạp, hoặc bạn muốn tách riêng phần định nghĩa template directive ra một file khác. không sử dụng nếu dùng template, nếu không directive sẽ ưu tiên sử dụng nội dung trong template.

VD:

replace

Replace nhận giá trị kỉểu boolean, mặc định là false.

  • True: Thay thế DOM element có khai báo directive bằng nội dung của directive tương ứng.
  • False: Đưa nội dung của directive vào bên trong DOM element khai báo directive đó.

VD:

transclude

Transclude nhận giá trị kỉểu boolean, mặc định là false.

  • True: Tách hết nội dung bên trong DOM element, sau đó thực thi directive, và cuối cùng là append hết phần nội dung đã tách vào DOM element.
  • False: Thay thế hết tất cả nội dung bên trong DOM element bằng nội dung của directive được khai báo.

Để dễ hiểu về AngularJS Directive, hãy xem qua tình huống sau: Giả sử trong tình huống bạn muốn hiển thị nhiều vùng dữ liệu có cùng format về giao diện, nhưng khác nhau về nội dung dữ liệu thì có nhiều cách để thực hiện.

  • Cách 1: Bạn có thể viết code ở những vị trí cần hiển thị. Cách này chắc chắn code sẽ rất thừa thải và không có khả năng tái sử dụng, rất khó khăn khi bạn muốn thay đổi format giao diện cho phần hiển thị đó vì phải tìm đến từng nơi.
  • Cách 2: Bạn có thể tạo một template chung và sử dụng javascript hay jQuery để đưa dữ liệu vào template mỗi khi gọi, nhưng cách này tương đối phức tạp, và xử lý tình huống này code sẽ không ít.
  • Cách 3: Bạn có thể sử dụng AngularJS và tạo một directive cho phần hiển thị đó. Cách này rất gọn gàng và tường minh, có thể tái sử dụng ở nhiều nơi, việc chỉnh sửa cũng dễ dàng hơn.

Tình huống trên đưa ra chỉ để dễ hình dung về directive, nhưng chưa nói lên đầy đủ ý nghĩa của directive. Bài viết này sẽ đưa ra những demo và giải thích để bạn hiểu rõ hơn.
